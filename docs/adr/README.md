# Tessera — Architecture Decision Records

Each ADR records one load-bearing decision: context, the decision, why, and consequences. They
correspond to the open-decision register in `tessera/docs/ROADMAP.md` (D1–D7). Status: Proposed /
Accepted / Superseded.

| [0038](0038-blob-opaque-preservation.md) | Blob block — opaque, bit-faithful preservation tier (`BlockKind::Blob`): store an un-parsed vendor file (Siemens `.l64`, GE `.7z`, PDF) **verbatim**, digest = `blake3(bytes)`, riding the existing seal/signature/verify unchanged; `format="blob"` (alias `junk`) ingest + `tessera extract`; FAIR F/A/R + integrity but **not Interoperable** until decoded — complements, never replaces, normalised array/table products | **Accepted** (as-built) |
| [0039](0039-compute-node-caching.md) | Compute-node caching — two tiers, both *infra not format* (#236): (A) lightweight = a Tessera versioning repo as the cache (`objects/<digest>` de-dups for free, `tessera gc` + atime-LRU sweep); (B) registry-grade = `zot` pull-through cache with `extensions.sync onDemand` + `storage.retention.policies.keepTags.pulledWithin` (access-recency, the predicate MinIO ILM lacks); standalone example flake + `nixosModules.tessera-cache`; one new in-scope verb `tessera get <ref>→path` (cache-agnostic fetch); explicit non-decision = caching is NOT a `tessera` subcommand | **Proposed** |
| [0040](0040-data-protection-and-confidentiality.md) | Data protection — schema-driven **sensitivity tiers** on `FieldSpec` (`public`/`coded`/`sensitive`/`identifying`, versioned + PS3.15-seeded; DICOM = a transitional special-case of the blob tier); consent-withdrawal **erasure** (destructive `redact` via forget/gc → lineage tombstone · crypto-shred via key-deletion for shared copies); **field-level confidentiality** (`age`/X25519 envelope-encrypt sensitive fields to recipient pubkeys → "coded/anon by default if leaked", sidecar to keep the seal deterministic; revisits ADR-0007 for the share/leak threat model) | Proposed |
| [0042](0042-self-contained-tsra-aux-members.md) | Self-contained `.tsra` — non-sealed `aux/` members (`.gitignore` for the seal): embed the ADR-0037 signature at `aux/signatures/<key_id>.sig.json` (default; `--sidecar` keeps the detached form) and stamp `aux/provenance.json` (RFC-3339 `ingested_at` + producer + host) at ingest. Load-bearing invariant: adding/removing `aux/` members leaves `content_hash` + `manifest_hash` byte-identical, so the conformance corpus is untouched and writer-determinism holds; wall-clock rides outside the seal by construction. Completes ADR-0037 §0 bug #2 (signature carriage) + closes #259 | Proposed |
| [0041](0041-provider-mediated-field-decryption.md) | Provider-mediated field decryption — **rung 2** of ADR-0040's confidentiality ladder (#241): per-tier DEK encrypts the field; **KEK held in an external provider** (KMS / Vault Transit) and accessed via an online `unwrap` handshake → time-gating + revocation + audit + **crypto-shred at scale** (the consent-withdrawal endgame that finally reaches leaked copies); recommends **unwrap-DEK over decrypt-as-a-service** (smaller compliance surface, lower latency, provider never sees PHI) and a **thin Tessera key-broker fronting a real KMS** (provider stays *infra*, not baked into the format — the `.tsra` carries only an opaque provider ref + wrapped-DEK + policy handle); reuses ADR-0040 §4's additive sidecar so writer-determinism is preserved (the seal still commits to plaintext / per-field commitment); **honest analog-hole limit** — provider gates key release + access over time, NOT what an authorized recipient does with decrypted plaintext (screenshot / memory dump / re-export); TEE/confidential-computing named as the partial rung 3 not in scope here | Proposed |
| [0043](0043-unified-recursive-hierarchy.md) | Unified recursive hierarchy (#215) — chunk-index (#214), multiscale pyramid + projections (#260), aux sidecars (#259/ADR-0042), collections (#223), composition/N-D (#216) are **the same node at different scales**: every level (chunk · block · pyramid-level · product · collection) carries a `{hash, stats, referencing}` triple, so verify/prune/summarise/reference are **one set of operations parameterised by scale**, not five bespoke paths; `{hash,stats}` are **byproducts of the streaming write** (fused pyramid + chunk-index + Merkle in one pass); draws the sealed-vs-`aux/` line the same way at every scale (ADR-0042). Synthesis ADR — no new format bytes; pins the invariant the children implement | Proposed |
| [0044](0044-sparse-data-representation.md) | Sparse data (#218) — pins the decision omnibus-recorded in ADR-0031: **COO table for scatter** (Vortex table with `shape`/`dtype`/`fill_value` metadata, canonical lex row order — reuses Merkle/monoid-stats/pushdown/SQL unchanged) · **dense-chunked array + `count=0` prune** for block-sparsity (free from ADR-0028) · **no bespoke sparse codec** (would fork integrity/stats/prune paths) · the lever is **materializability + access pattern**, not occupancy % (bench #221-A) · single-linear-index emit is built, multi-axis emit is a documented extension (gap) | Proposed |
| [0045](0045-units-and-quantities.md) | Units & quantities (#219) — pins the units half of ADR-0032: **UCUM** codes on `ArraySpec.unit`/`FieldSpec.unit` + `_vocabulary`/`_code` escape for domain codes (DICOM/RadLex/SNOMED) · **quantity = raw storage + `(rescale_slope, rescale_intercept, unit)` transform**, physical recovered on read (store-don't-compute; lossless ints round-trip; CT→HU, PET→Bq/mL) · derived physical quantities (SUV, ADC) are **products** with `derived_from` edges, not descriptors · no in-format units engine — hand off to a downstream UCUM library · **gaps:** UCUM validation not enforced (needs `--strict-units` lint), coded-vocabulary values not cross-checked (`--meta coded` lint) | Proposed |
| [0046](0046-time-model.md) | Time model (#220) — pins the time half of ADR-0032: **RFC-3339 UTC** for `Manifest.timestamp` (identity-key, sealed) and `aux/provenance.json:ingested_at` (ADR-0042, outside the seal — writer-determinism) · **elapsed = quantity** via `Referenced::time_regular/time_irregular` (unit `s`, frame `epoch:<instant>`) · **event times = integer ticks + scale** via `time_ticks(period_s, start_s)` (lossless, compact, no float drift — closes the review's "listmode `ms` column has no described semantics" gap) · absolute instant = elapsed + `epoch:<instant>`; canonical epochs `unix`/`acquisition_start`/`injection` · monotonic for intervals, UTC only for stamps · **gaps:** listmode schema not yet carrying the descriptor · cross-block epoch coherence not verified · UTC anchor for `epoch:<instant>` resolutions not yet pinned in the vocabulary | Proposed |
