**evidence** (spike/bench/test). Future development is benched/compared against this вАФ perf rows
are *regression floors* (don't go below), correctness rows are *required* (binary).

## D. Performance SLA gates (regression floors вАФ benched, 88-core box)
Rust benches: `cargo bench -p tessera-io` (`benches/codec.rs`). Wall-clock floors are machine-dependent
(measured below, **not** CI-gated); the machine-independent compression-ratio floor **is** gated
(`array::tests::pcodec_compresses_smooth_int16_volume`). Python-spike numbers retained where the bench
needs real CT/DICOM. Cross-substrate comparison vs the bare backends (`.tsra` overhead, size ladder,
ROI/slice speedups): `cargo run -p tessera-io --example bench_compare --release` вЖТ SPIKE-RESULTS.md #143.

**`.tsra` container tax vs the bare codec** (same 128¬≥ int16, 8√Ч 64¬≥ chunks, same run вАФ `tsra_vs_bare`):
size = bare codec bytes + ~320вАУ600 B zip + ~1 KB FAIR manifest (вЙ™1% at scale); **write +0.7%** (zip
STORE + blake3 seal); **read +11%** (the seal + per-block digest *verification* vanilla skips вАФ the
integrity it buys); **3-D ROI 32¬≥-of-128¬≥ вЙИ 2.3√Ч faster than full decode** (cubic chunks; only the
intersecting chunk is read). A timed cross-*ecosystem* run (vs zarr-python/vortex-python on DUPLET
data) is the remaining dedicated harness (#143).
